개선 사항
1. 괄호를 포함한 사칙연산 구현. 
2. 변수와 함수들이 하나의 역할만 하도록 구현.
3. 

버그 수정

버그
1. 결과로 음수가 출력되면 그 다음 연산에서 값이 0으로 고정. 연산 과정에서 switch문에 case'-'가 있어서 발생하는 버그
2. 몇몇 테스트케이스 통과 못함 - 구조를 짜고 코딩 합시다.

요청 사항
1. 버튼을 누르면 구별해 줄 함수 선언 - OK
2. 숫자인지 연산자인지 구분 및 Control할 함수 - 기능은 구현. 함수로 만들 예정.
3. Control함수에서 연산한 데이터가 쌓이는 Data Class 선언 - 아직 클래스 설계 안했음.ㅇ
4. Data Class 데이터를 저장할 Set, Get 함수 선언 
5. UI에 Display할 때 Data Class에 저장한 멤버변수 Get해서 표시 - 현재 컨트롤의 연산결과를 Get으로 가져와 디스플레이 하는 기능 구현.
6. 변수와 함수, 클래스는 반드시 하나의 역할만 수행한다. 
7. 라디오 버튼을 사용해 리스트박스 활성화시키기. 
8. 리스트박스에 연산 히스토리 디스플레이하기.
